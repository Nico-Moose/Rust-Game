# Progression and Balance

## Design goal
Keep the emotional curve of Rust while removing excessive downtime and grind.

## Early / Mid / Late structure

### Early game
**Target length:** first 1–3 sessions.

**Player goals:**
- gather starter resources
- make primitive tools
- place a starter base
- survive first PvP encounters
- get first scrap income

**Expected feeling:** vulnerable, hungry, hopeful.

**Balance principles:**
- fast starter base setup
- quick primitive crafting
- limited punishment for first few deaths
- low-cost rebuild path

### Mid game
**Target length:** most of the season.

**Player goals:**
- farm scrap and components
- unlock workbench tier 2
- reinforce the base
- participate in monument runs
- build team coordination or solo efficiency
- begin small-scale raiding pressure or counter-raiding

**Expected feeling:** tense, strategic, opportunistic.

**Balance principles:**
- this should be the richest content layer
- meaningful tradeoffs between PvP and farming
- monuments should become conflict magnets
- progression should feel earned but not glacial

### Late game
**Target length:** final stage before wipe, rarer than mid game.

**Player goals:**
- unlock top gear and defenses
- control premium zones
- perform serious raids
- dominate or protect territory
- compete in seasonal leaderboards

**Expected feeling:** powerful, paranoid, expensive.

**Balance principles:**
- not everyone reaches full dominance quickly
- late game requires coordination and risk
- raids are expensive and visible
- upkeep and exposure prevent unchecked mega-empires

## Fast-session balancing for mini app
### What gets shortened versus Rust
- resource collection time
- starter base time
- average travel time
- menu friction
- amount of junk loot

### What stays intense
- loot loss on death
- monument contesting
- raid preparation cost
- defense pressure
- seasonal race

## Example balancing rules
- A useful early farming run should fit into 5–10 minutes.
- A player should be able to build a tiny secure starter within the first session.
- First tech unlock should happen quickly enough to create momentum.
- Mid-tier monuments should become worthwhile within the first day of serious play.
- Raiding should remain a costly, late-season aspiration for many players.

## Solo vs group balance
### Solo needs
- cheap hidden base options
- efficient low-profile farming paths
- reduced setup friction
- strong fallback progression even after losses

### Group advantages
- faster monument control
- better raid power
- stronger defense rotations
- safer transport and loot extraction

### Anti-zerg checks
- higher upkeep for huge bases
- vulnerability through visibility
- contested event design that can punish overextension
- diminishing returns for giant storage compounds

## Retention balance
The player should always leave a session thinking one of these:
- "I came back with profit."
- "I died, but I know what to do next."
- "My base is stronger now."
- "I am close to my next unlock."
- "Next session I can contest a better zone."
