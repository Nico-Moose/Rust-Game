# Game Design Document

## 1. Core fantasy
You are a survivor in a brutal post-industrial wasteland where every run can make or break your progress. You gather resources, craft tools, build a defensive shelter, risk trips into hostile monuments, clash with rival players, and grow from a primitive scavenger into a dangerous faction or solo raider.

## 2. Core gameplay loop
1. Spawn with minimal gear.
2. Gather nearby resources.
3. Craft primitive tools and basic healing.
4. Build a small starter base with a door and storage.
5. Go on loot runs into monuments.
6. Return safely with scrap, components, and materials.
7. Upgrade workbench and unlock tech.
8. Improve base defenses and upkeep stability.
9. Fight for territory, loot, and map control.
10. Prepare for raids or repel them.
11. Repeat until wipe, then start the next season with new goals.

## 3. Session structure
The game is designed for compact but intense sessions.
- **5 minutes**: harvest, craft, deposit, small route.
- **10 minutes**: monument loop, trade, tech unlock.
- **15–20 minutes**: PvP hunt, team run, defense, event contest.

## 4. Player state
Each player has:
- Twitch-linked identity
- Inventory
- Equipped gear
- Health
- Hunger
- Energy or stamina
- Radiation or contamination exposure in special zones
- Home/base ownership state
- Tech progress
- Season statistics

## 5. Player lifecycle
### Early game
- Spawn nearly naked
- Gather wood, stone, cloth, food
- Craft hatchet, pickaxe, spear or bow
- Place a starter base
- Secure first loot chest and cupboard/core

### Mid game
- Farm scrap and components
- Unlock workbench tier 2
- Improve building materials
- Enter higher-risk monuments
- Use ranged weapons regularly
- Begin organized defense and opportunistic PvP

### Late game
- Access top monuments and rare resources
- Build fortified compounds
- Deploy traps, turrets, generators, advanced locks
- Craft premium weapons and raid tools
- Control territory and run large-scale raids

## 6. Death and loss
Death is meaningful.
- The player drops carried loot on death.
- Other players can loot the body.
- The killer gains momentum; the victim loses time and resources.
- Respawn happens at a sleeping point, shelter, or fallback spawn.

## 7. Seasonal loop
Seasons or wipes reset progression on a fixed cadence. Wipes keep the economy healthy, prevent long-term runaway dominance, and re-open the race for everyone. Typical target: every 2–4 weeks.

## 8. Map logic
The map is compact, readable, and dense with points of conflict.
- Safe hub(s)
- Low-risk roadside routes
- Mid-risk resource zones
- High-risk elite monuments
- World events
- Buildable land parcels or open building sectors

## 9. Success metrics
The design succeeds if players regularly feel:
- tension when leaving base
- relief when returning with loot
- attachment to their base
- visible progression after short sessions
- meaningful rivalry and map stories
- anticipation for wipes and seasonal rewards
